## Table_14 标桥BP统计

> 脚本：`workflow_step_data_scripts/table_14_biaoqiao_bp.py`

### 数据源文件

| 文件 | 匹配模式 | 说明 |
|---|---|---|
| `Data/{year}{month}/source_data/营收平台标桥收益数据.xlsx` | 所有 sheet | 标桥收益数据，遍历所有 sheet 按分公司累加收益 |

| `persistence_data/分公司映射表.xlsx` | Sheet 0 | 分公司名称映射（源表名称 → 月报输出名称） |
| `Data/{year}{month}/process_data/extract_data{prior_month}月报.xlsx` | Sheet「表格14」 | 上期数据，用于读取各分公司上月收益和上月全年收益 |

### 逐列统计方式

| # | 列名 | 数据来源 | 统计逻辑 |
|---|---|---|---|
| 1 | 序 | 生成 | 从 1 开始的自增序号 |
| 2 | 分公司 | 标桥_bp → `分公司` | 按 BP 表顺序列出，决定结果行顺序 |
| 3 | 本月收益（元） | 营收平台标桥收益数据 | 遍历所有 sheet，按分公司累加 `收益金额（元）` 或 `收益（元）` 列，经映射表统一名称后匹配 |
| 4 | 上月收益（元） | 上期 extract「表格14」 | 按分公司名匹配上期的 `本月收益（元）` |
| 5 | 环比变化 | 计算值 | `(本月收益 - 上月收益) / 上月收益`，正数前缀 ▲，负数前缀 ▼ |
| 6 | 同比变化 | — | 暂留空（标记为 `/`） |
| 7 | 全年收益（元） | 上期 extract + 本月收益 | 报告月为 1 月时 = 本月收益（新年重置）；其他月份 = 上期全年收益 + 本月收益（滚动累加） |
| 8 | BP总额（元） | 标桥_bp → `BP总额（元）` | 按分公司名匹配 |
| 9 | BP完成率 | 计算值 | `全年收益 / BP总额`，格式化为百分比（如 `41.17%`） |

### 数据流

```
营收平台标桥收益数据.xlsx ──┐
标桥_bp.xlsx ───────────────┤
分公司映射表.xlsx ───────────┤──→ 名称映射 + 收益累加 + BP 拼接 ──→ 输出 Table_14
上期 extract_data{month}月报.xlsx ┘                                      ↓
                                                              保存到 extract_data{month}月报.xlsx「表格14」
```

### 注意点
- 结果行顺序严格按 BP 表顺序，不按收益排序
- 收益数据遍历源文件所有 sheet，按分公司累加（兼容 `收益金额（元）` 和 `收益（元）` 两种列名）
- 分公司名称通过映射表统一：源表中的名称映射为月报输出名称，未匹配的保留原名
- 全年收益采用滚动累加，但 1 月时重置为本月收益（新年开始，不累加去年全年）
- 同比变化字段暂未实现，固定输出 `/`
- 上期 extract 文件路径：`Data/{year}{month}/process_data/extract_data{prior_month}月报.xlsx`（在报告月目录下）

## Table_19 投标保函-分公司收益

> 脚本：`workflow_step_data_scripts/table_19_baohan_bp.py`

### 数据源文件

| 文件 | 匹配模式 | 说明 |
|---|---|---|
| `Data/{year}{month}/source_data/数据服务收益明细表.xlsx` | Sheet 0（26年数据） | 当年数据服务收益明细，筛选 `模块=电子投标保函平台（营运）` |
| `Data/{year}{month}/source_data/数据服务收益明细表.xlsx` | Sheet 1（25年数据） | 去年同期数据，用于同比计算 |
| `persistence_data/保函_bp.xlsx` | Sheet 0 | 各分公司 BP 总额 |
| `Data/{prior_year}{prior_month}/process_data/extract_data{prior_month}月报.xlsx` | Sheet「表格19」 | 上期数据，用于读取各分公司上月收益 |

### 逐列统计方式

| # | 列名 | 数据来源 | 统计逻辑 |
|---|---|---|---|
| 1 | 序 | 生成 | 从 1 开始的自增序号 |
| 2 | 分公司 | 数据服务收益明细表 → `分公司` | 经 `normalize_branch()` 统一后的分公司名，取所有出现过的分公司并集 |
| 3 | 本月收益（元） | 26年数据 | 筛选 `月 == month`，按分公司对 `销售毛利(元)` 求和，保留 2 位小数 |
| 4 | 上月收益（元） | 26年数据 | 筛选 `月 == month-1`，按分公司对 `销售毛利(元)` 求和 |
| 5 | 环比变化 | 计算值 | `(本月 - 上月) / 上月` |
| 6 | 同比变化 | 计算值 | `(本月 - 去年同月) / 去年同月`，基于 25 年同月数据 |
| 7 | 全年收益（元） | 26年数据 | 筛选 `月 <= month`，按分公司对 `销售毛利(元)` 求和，保留 2 位小数 |
| 8 | BP总额（元） | 保函_bp | 按分公司名匹配（经 `normalize_branch()` 统一后） |
| 9 | BP完成率 | 计算值 | `全年收益 / BP总额`，格式化为百分比 |

### 数据流

```
数据服务收益明细表.xlsx（26年）──→ 按模块筛选 + 按分公司/月份聚合 ──┐
数据服务收益明细表.xlsx（25年）──→ 同比数据 ────────────────────────┤
保函_bp.xlsx ───────────────────────────────────────────────────────┤──→ 输出 Table_19
上期 extract_data{month}月报.xlsx ──────────────────────────────────┘            ↓
                                                                      保存到 process_data_result.xlsx「投标保函-分公司收益_1」
                                                                      保存到 extract_data{month}月报.xlsx「表格19」
```

### 注意点
- 数据源按 `年` 列分为两个 sheet（sheet 0 = 26年，sheet 1 = 25年），通过 `engine='calamine'` 读取
- 筛选条件为模块包含 `电子投标保函平台（营运）`
- 分公司名称有独立的 `normalize_branch()` 实现（不使用 utils 中的版本），处理了如 `兰州分公司-交易 → 兰州分公司` 等特殊映射
- 分公司取并集：本月、上月、全年任一有数据的分公司都会出现在结果中
- 同比变化使用 `calculate_huanbi()` 函数计算，与环比使用同一逻辑
